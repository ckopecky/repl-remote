---
name: Orval generated package deep imports
description: Why deep-importing generated files from an Orval-generated hooks package fails even though the file physically exists.
---

Packages generated by the `api-client-react` / `api-zod` codegen workflow declare a
single `"."` entry in `package.json` `exports`, pointing at `src/index.ts`. That index
re-exports everything from `./generated/api` and `./generated/api.schemas`.

Importing a deep path like `@workspace/api-client-react/src/generated/api.schemas`
fails at Vite/esbuild resolution time ("Missing ... specifier in package") because
Node's `exports` map blocks any path not explicitly listed, regardless of whether the
file exists on disk.

**How to apply:** always import generated types/enums/schemas from the package root
(`@workspace/api-client-react`, `@workspace/api-zod`), never via a `/src/...` or
`/generated/...` subpath. If a design subagent or generated frontend code produces a
deep import like this, it's a quick, mechanical fix — just flatten the import to the
package root.
